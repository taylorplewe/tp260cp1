The 6502 assembly language consists of 16 different addressing modes and 69 different instructions.

The most frequently used instructions the programmer uses are lda, which loads the accumulator with whatever comes afterwards, and sta, which stores the value of the accumulator at the given address. These two isntructions (as well as ldx, stx, ldy and sty) prove more than essential in any 6502 program.

As an example, if I wanted to put the value 9 at memory location $0320, I would write the code

lda #$09

sta $0320

$ signifies the number that comes after it is hexadecimal. % means binary, and nothing means decimal.

Of course, the value 9 and memory location 0x0320 would both mean something in context. To help with remembering those contexts, the programmer can assign labels to number values, for instance:

COLOR_RED = 9

box_1_color = $0320

And then the code would be much more human readable:

lda #COLOR_RED

sta box_1_color

Now say I wanted to make box_1 quickly alternate between 256 different colors. I could do something like this:

ldx #COLOR_RED

.loop:

stx box_1_color

inx

jmp .loop

inx means increase the value of X, and jmp just tells the computer to jump to the given location.

As you might expect, that's a big generalization of how actual functioning code might look; I don't think there's any computer out there that would turn just that into something nice to look at. This is just a basic look at what 6502 assembly looks like.
